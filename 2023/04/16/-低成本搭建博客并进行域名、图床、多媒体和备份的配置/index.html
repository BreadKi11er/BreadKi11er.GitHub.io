低成本搭建博客并进行域名、图床、多媒体和备份的配置

2023.4.13

包括 GitHub Pages 在内的工具日新月异，加之网络上大部分教程面向对象主要为程序员，所以我希望通过本文对教程中较为含糊的步骤进行解释，以帮助小白。本文以这篇文章为基础进行 GitHub Pages + hexo +Fluid 主题的博客的搭建，「低成本」指的是这个搭建组合唯一需要支出的费用仅为域名费用，不购买域名就是免费，但我觉得「没内味」。本文主要记录搭建过程中遇到的一些问题以及解决方法。

博客搭建

  1. 在安装 Git 的时候没有进行邮箱的配置，似乎没有问题。

  2. 在建立新的仓库（repository）的时候，注意要选用 Public，这个是默认的，即只要填上 <用户名>.github.io 就可以了。

  3. 如下修改 Hexo 博客目录中的 _config.yml：
    theme: fluid # 指定主题
    language: zh-CN # 指定语言，会影响主题显示的语言，按需修改

    • 注意这一步以及下面所有需要打开文件进行修改的步骤，都建议通过 VS Code 文本编辑器来进行。
    • theme 实际上就是储存在 hexo 安装位置 C:\Users\<用户名>\hexo-blog 里的 themes 中的文件夹，这也是为什么上一步需要把解压后的文件夹改名。
    • language 可以通过 C:\Users\<用户名>\hexo-blog\themes\fluid\languages 中的文件进行查询。如果希望 Fluid 使用台湾繁体那就改为 zh-TW。

  4. 第九步发布到 GitHub Pages中，要获取 repo 的链接，在你的仓库的首页点击 Code 后复制 HTTPS 下的链接即可。

图床设置

我已有一台黑群晖，但担忧于它的稳定性，我选择使用 PicGo + GitHub + Jsdelivr 的组合。根据这篇教程设置，一路顺利，唯一的小问题的是 PicGo 在托盘中需要双击才能打开。

Fluid 主题配置

官方文档很详尽，只记录几点。

  • 在 cd hexo-blog 后用 hexo g 生成静态页面。当更新了 _config.fluid.yml 后，需要使用它重新生成静态页面，才能使更改生效，不过只是修改文章内容是不需要的，保持 http://localhost:4000/ 在线，刷新后就可更新。而 hexo s 是指启动本地服务器。在执行这个命令后，可以通过本地浏览器访问 http://localhost:4000/ 来查看当前 Hexo 博客的效果。这样可以在本地预览、测试博客主题、排版等效果，而不必每次修改后都上传到服务器上才能查看效果，在完成所有修改后，用 hexo g -d 来上传到服务器，这时候域名所在网站就会部署完成。

  • 字体家族根据这篇文章进行设置 font-family: "Helvetica Neue", Helvetica, Arial, "PingFang SC", "Hiragino Sans GB", "Heiti SC", "Microsoft YaHei", "WenQuanYi Micro Hei", sans-serif
    但是这样最终效果是，中文会使用微软雅黑，虽然这个字体渲染得很好看，但我已经看腻了。在删除了 Microsoft YaHei 之后，网站会使用华康黑体简 W5-A 这个字体，而使用 F12 => Ctrl+Shift+M 模拟移动设备进行阅览时仍然为这个字体。我喜欢这个字体较短的中文引号（「」）但不喜欢它方正的分号和逗号（； ，），不过对我来说只要不是微软雅黑，基本都能接受。注意在设置字体家族时，末尾不能有分号，直接复制粘贴知乎文末的代码， CMD 中会报错：err: YAMLException: can not read a block mapping entry，同样地，如果在修改 _config.fluid.yml 并进行 hexo g => hexo s => http://localhost:4000 后发现页面并没有变化，观察 CMD 中的错误可以帮助解决问题。也建议单引号将每个字体系列括起来，并在外面再用双引号将它们封装成一个字符串，以下为我最终的设置：
    font_family: "'Helvetica Neue', 'Helvetica', 'Arial', 'Hiragino Sans GB', 'Heiti SC', 'WenQuanYi Micro Hei', sans-serif"。
