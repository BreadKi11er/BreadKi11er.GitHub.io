[完整步骤]解决群晖套件 Transmission「too many open file」导致的红种

2023.3.28
在 DSM 7.01 系统中操作。

原文

  1. 用 SSH 登录群晖并给予 root 权限
    DSM 官方手册
    中提到的

    命令：ssh DSM/SRM 管理员帐户@DSM/SRM IP 地址 -p SSH 端口号

    示例：ssh myadmin@10.17.2.6 -p 22

    端口号可以在 控制面板-终端机和 SNMP-终端机中找到。同时你需要确保启动 SSH 功能打了勾。

    把群晖密码复制粘贴到 password 中，不显示密码是正常的，粘贴完直接回车即可。看到后 myadmin@DiskStation:/$ 再输入 sudo -i并再次粘贴密码获取 root权限。完成后应该会显示 root@DiskStation:~#。

  2. 下载原文提供的文件后，上传到群晖中，复制该文件所在目录的地址。

  3. 使用 cd "粘贴所在目录地址" 定位到该文件所在目录。如果不加上英文双引号，可能会报错 -ash: cd: too many arguments。

  4. 输入 chmod 755 rlimit 来给该文件提供权限。

  5. 输入 ps -ef | grep trans 获取 Transmission 的进程编号。

在输出中找到如上述 17068 进程编号，即为Transmission 的进程编号。

  1. 输入 ./rlimit 进程编号 就可开始更改。
  2. 输入 cat /proc/进程编号/limits 查看是否完成操作。

看到如上图所示即为修改成功。
